🏥 HMS Admin Workflow Specification
1. Admin Login & System Access
Goal: Secure, authenticated entry with immediate role validation.

Authentication Flow
Input: Username/Email + Password.
Validation:
Check credentials (bcrypt hash comparison).
CRITICAL: Verify is_active = true status.
CRITICAL: Verify Role exists in roles table.
Token Generation: Issue JWT containing userId, roleId, branchId, and permissions array.
Role Enforcement
If Role != ADMIN (and system is in Admin Mode), deny access immediately.
Redirect to Admin Dashboard upon success.
Session Management
Auto-logout on 15 minutes of inactivity for security.
Force password change for first-time login.
2. Admin Dashboard (Command Center)
Goal: A high-level cockpit for operational awareness. No data entry here, only monitoring.

Real-Time KPI Widgets
Revenue Ticker: Today's Collection (Cash vs. Insurance/Digital).
Patient Census: Currently admitted (IPD) vs. Today's walk-ins (OPD).
Staff Active: Count of Doctors/Nurses currently clocked in.
Critical Alerts: Pharmacy stock < 10% items, Lab delays > 2 hours.
Quick Action Hub
New User Registration.
Global Patient Search.
Approve Pending Discounts/Refunds.
3. Hospital & Branch Management
Goal: Define the legal and physical entity of the hospital.

Hospital Profile Configuration
Setup: Input Hospital Name, Logo, Address, Tax ID (GST/VAT).
Impact: These details auto-populate on Bill Definitions and Report Headers.
Department Management
Action: Create/Edit Departments (e.g., Cardiology, Orthopedics).
Linkage: Link Departments to specific Floors or Wards.
Branch/Location Control (If Multi-Branch)
Action: Create Branch -> Assign Branch Admin.
Data Scope: Admin toggles between "Global View" (All Branches) or "Local View" (Specific Branch).
4. User & Role Management
Goal: Strict RBAC (Role-Based Access Control) administration.

Role Definition (The "What")
Core Roles: Admin, Doctor, Nurse, Receptionist, Pharmacist, Lab Technician.
Permission Matrix: Admin checks boxes for each role (e.g., Doctor: write_prescription: true, view_revenue: false).
User Onboarding (The "Who")
Flow: Create User → Assign Role → Assign primary Department.
Security: Generate temporary password (email to user).
Lifecycle Management
Deactivation: "Soft Delete" staff who leave (never hard delete to preserve audit history).
Lockout: Auto-lock account after 5 failed attempts.
5. Patient Management Oversight
Goal: Data integrity and exception handling. Admin does not do daily registration.

Data Governance
Merge Records: Tool to merge duplicate patient files (e.g., "John Doe" and "Jon Doe").
** Correction:** Edit locked fields (e.g., DOB, Gender) that Receptionists cannot change after 24 hours.
Access Control
VIP Flagging: Mark generic profiles as VIP (restricts visibility to Senior Doctors/Admin only).
Discharge Override: Force discharge a patient if the system flow gets stuck.
6. Lab Master & Diagnostic Control
Goal: Define the service menu and operational standards.

Test Master Configuration
Step 1: Define Test Name (e.g., "CBC").
Step 2: Define Parameters (e.g., "Hemoglobin", "WBC").
Step 3: Set Reference Ranges (Male/Female/Child/Newborn).
Step 4: Set Price & Turnaround Time (TAT).
Operational Monitoring
Queue Watch: View list of samples collected but results pending > TAT.
Outsource Mgmt: Flag tests that are sent to external labs.
7. Pharmacy & Inventory Governance
Goal: Financial control over stock and safety.

Medicine Master (Formulary)
Action: Add Drug Generic Name, Brand Name, Dosage Form (Tablet/Syrup).
Safety: Flag high-risk drugs (Narcotics) requiring double-approval for dispensing.
Stock & Procurement
Re-order Levels: Set threshold (e.g., < 50 units). System triggers alert.
Supplier Mgmt: Link suppliers to specific drug categories.
Inventory Audit: Periodic "Physical vs. System" stock adjustment (Requires Admin Reason Code).
8. Billing, Revenue & Financial Oversight
Goal: Revenue assurance and fraud prevention.

Service Rate Master
Action: Define prices for Consultations, Bed Charges, Surgeries.
Rules: Set differential pricing (e.g., General Ward vs. Private Room).
Exception Handling
Refunds: Reception requests refund → Admin Dashboard Notification → Admin Approves/Rejects.
Discounts: Define max discount % per role. Admin approves anything above limit.
End-of-Day (EOD) Closure
Workflow: Admin reviews "Cash in Hand" vs. "System Total" → Signs off digital "Day Close".
9. Reports & Analytics
Goal: Data-driven decision making.

Financial Reports
Daily Collection Report: Breakdown by Cash/Card/UPI/Insurance.
Pending Dues: List of IPD patients with outstanding balances > limit.
Clinical/Operational Reports
Doctor Productivity: Patient count per doctor per day.
Lab Turnaround: % of tests completed within defined TAT.
Export Engine
Generate PDF for official filing.
Export CSV/Excel for external accounting (Tally/QuickBooks).
10. System Configuration & Settings
Goal: Technical application control.

Global Settings
DateFormat: Set DD/MM/YYYY or MM/DD/YYYY.
Currency: Set symbol (₹, $, €).
Notifications: Toggle SMS/Email gateways on/off.
Scheduler Configuration
Set Backup time (e.g., 2:00 AM Daily).
Set Auto-discharge limits.
11. Audit Logs & Compliance
Goal: Legal security and "Who did what".

Activity Logger
Every POST, PUT, DELETE request is logged with timestamp, IP, User, OldValue, NewValue.
Searchable History
Scenario: "Who changed the price of MRI Scan?"
Action: Filter Logs by Module: "Billing" → Entity: "Service Master".
12. End-to-End Admin Workflow Summary
(A typical day for the HMS Admin)

08:00 AM: Login. Check Dashboard for critical stock alerts or overnight IT issues.
09:00 AM: Review User Mgmt. Activate accounts for new hires. Deactivate outgoing staff.
11:00 AM: Financial Oversight. Approve high-value refunds or discount requests from the billing counter.
02:00 PM: Master Updates. Update prices for Lab Tests or add new Medicine stock entries based on purchase invoices.
05:00 PM: Audit. Check "Voided Bills" report to ensure no fraud (cash taken, bill cancelled).
07:00 PM: Reporting. Generate "Daily Collection Report" & "Census Report".
07:30 PM: Trigger Day Close (freezes financials for the date). Logout.
